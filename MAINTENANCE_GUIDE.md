## 🚀 2. بروتوكول النشر بـ "ضغطة زر" (The Golden Deployment)

يُمنع النشر اليدوي أو رفع الملفات بشكل عشوائي. يتم التحديث من بيئة التطوير إلى الإنتاج عبر هذه السلسلة المتكاملة:

### الخطوة الأولى: في بيئة التطوير (Local)
قم بتوثيق التغييرات ورفعها إلى المستودع:
```bash
git add .
git commit -m "Summary of changes"
git push origin main
```

### الخطوة الثانية: الرفع للسيرفر (One-Click Update)
تأكد أن السيرفر يعمل بنسخة **Node v20 (LTS)** كما هو الحال الآن. نفذ هذا الأمر الموحد:
```bash
ssh root@165.232.127.15 "cd /root/talabia-app && git fetch origin main && git reset --hard origin/main && chmod +x deploy.sh && ./deploy.sh"
```
**ماذا يفعل هذا الأمر؟**
1. يدخل للمجلد الموحد.
2. يسحب أحدث كود من GitHub.
3. ينظف البيئة من أي ملفات مؤقتة.
4. يقوم ببناء (Build) الواجهة الأمامية وتحديثها آلياً.
5. يعيد تشغيل الخدمات لضمان سريان التعديلات فوراً.

## ⚙️ 3. إدارة العمليات (Backend Management)

يعمل النظام في الخلفية عبر **PM2**. القواعد التنظيمية هي:
- **الاسم الرسمي**: `talabia-prod-backend` (ولا يُسمح بتكراره بأسماء أخرى).
- **التشغيل التلقائي**: عند حدوث أي خطأ، يقوم PM2 بإعادة التشغيل آلياً.
- **الحفظ**: بعد أي تعديل في الإعدادات، يجب تنفيذ `pm2 save` لضمان الاستقرار بعد إعادة تشغيل السيرفر.

---

## 🎨 4. معايير الجودة (The Golden Standard)

للحفاظ على المظهر الاحترافي ومنع "الارتباك البصري":
- **الصور**: يجب أن تكون دائماً بنسبة أبعاد `4:5` وتستخدم `object-fit: cover`.
- **النصوص**: محاذاة العناوين والوصف يجب أن تكون دقيقة بحد أقصى سطرين (Line-clamp).
- **المجلدات**: يُمنع إنشاء مجلدات جديدة خارج الهيكلية المعتمدة.

---

## 🚫 5. تجنب هذه الأخطاء (Anti-Patterns)

- ❌ **لا تعدل مباشرة** على الملفات داخل `/var/www/`؛ التعديل يبدأ من المستودع دائماً.
- ❌ **لا تكرر الملفات**؛ استخدم المسارات النسبية الموحدة.
- ❌ **لا ترفع صوراً** ذات أحجام ضخمة دون ضغطها.
- ❌ **لا تترك ملفات `.bak` أو `.old`**؛ السيرفر يجب أن يظل "نظيفاً" تماماً.
