## 🚀 2. بروتوكول النشر بـ "ضغطة زر" (The Golden Deployment)

يُمنع النشر اليدوي أو رفع الملفات بشكل عشوائي. يتم التحديث من بيئة التطوير إلى الإنتاج عبر هذه السلسلة المتكاملة:

### الخطوة الأولى: في بيئة التطوير (Local)
قم بتوثيق التغييرات ورفعها إلى المستودع:
```bash
git add .
git commit -m "Summary of changes"
git push origin main
```

### الخطوة الثانية: الرفع للسيرفر (One-Click Update)
تأكد أن السيرفر يعمل بنسخة **Node v20 (LTS)** كما هو الحال الآن. نفذ هذا الأمر الموحد:
```bash
ssh root@165.232.127.15 "cd /root/talabia-app && git fetch origin main && git reset --hard origin/main && chmod +x deploy.sh && ./deploy.sh"
```
**ماذا يفعل هذا الأمر؟**
1. يدخل للمجلد الموحد.
2. يسحب أحدث كود من GitHub.
3. ينظف البيئة من أي ملفات مؤقتة.
4. يقوم ببناء (Build) الواجهة الأمامية وتحديثها آلياً.
5. يعيد تشغيل الخدمات لضمان سريان التعديلات فوراً.

## ⚙️ 3. إدارة العمليات (Backend Management)

يعمل النظام في الخلفية عبر **PM2**. القواعد التنظيمية هي:
- **الاسم الرسمي**: `talabia-prod-backend` (ولا يُسمح بتكراره بأسماء أخرى).
- **التشغيل التلقائي**: عند حدوث أي خطأ، يقوم PM2 بإعادة التشغيل آلياً.
- **الحفظ**: بعد أي تعديل في الإعدادات، يجب تنفيذ `pm2 save` لضمان الاستقرار بعد إعادة تشغيل السيرفر.

## 🎨 4. معايير الجودة (The Golden Standard)

للحفاظ على المظهر الاحترافي ومنع "الارتباك البصري":
- **الصور**: يجب أن تكون دائماً بنسبة أبعاد `4:5` وتستخدم `object-fit: fill`.
- **التخطيط**: يُحظر استخدام الهوامش السالبة (`negative margins`) لتعويض الفراغات الناتجة عن الـ `main` padding؛ استخدم دائماً بروتوكول التثبيت الموضح أدناه.
- **النصوص**: محاذاة العناوين والوصف يجب أن تكون دقيقة بحد أقصى سطرين (Line-clamp).

## 🏗️ 6. معايير تخطيط الواجهة (UI Layout & Docking Protocol)

للمحافظة على مظهر "لوحة التحكم" ملتصقة بالـ Navbar بدون فراغات (Gaps):

1. **بروتوكول التثبيت (Fixed Immersion)**:
   - يجب أن يكون الهيدر العلوي لأي لوحة تحكم (بائع، مدير، شركة شحن) بوضعية `position: fixed` وليس `sticky`.
   - الإحداثي العلوي (`top`) يجب أن يُحسب بدقة لمطابقة نهاية الـ Navbar:
     `top: calc(var(--navbar-height) + var(--safe-inset-top));`
   
2. **بروتوكول الفراغ التعويضي الديناميكي (The Dynamic Spacer Rule)**:
   - بما أن الهيدر مثبت (`fixed`) ويخرج من نطاق التوزيع، يجب وضع `div` فارغ (Spacer) يليه مباشرة في كود الـ JSX.
   - **يُحظر استخدام الارتفاعات الثابتة (Static Heights)**: لأن الهيدر قد يتمدد رأسياً بسبب التفاف النصوص (مثل اسم المتجر الطويل) أو عند العرض على الجوال.
   - **الحل الإلزامي**: استخدام `ResizeObserver` في React لمراقبة الارتفاع الفعلي للهيدر (`offsetHeight`) وتحديث ارتفاع الـ `Spacer` برمجياً بشكل لحظي لضمان عدم اختفاء أي جزء من محتوى الصفحة (مثل هيدر المحفظة) تحت هيدر لوحة التحكم.

3. **التحكم الشامل (Horizontal Anchoring)**:
   - يجب ضبط `left: 0; right: 0; width: 100%;` للهيدر المثبت لضمان عدم تأثره بأي بادنج جانبي للحاويات الأب.

## 🔐 7. معايير التنقل والجلسة (UX & Auth Protocols)

لضمان تقديم تجربة "تطبيق تجاري" (Commercial-Grade App) سلسة وموثوقة كالشركات الكبرى، يُلزم الالتزام التام بهذه المعايير:

1. **الاستعادة الموحدة للتمرير (Unified Scroll Restoration)**:
   - نظرًا لمعمارية التطبيق، التمرير يحدث داخل حاوية `<main id="main-scroll-container">` وليس في نافذة المتصفح الرئيسية.
   - ❌ **الخطأ**: استخدام `window.scrollTo(0,0)` عند تبديل الصفحات، مما يجعل المستخدم يعلق في وسط الصفحة الجديدة.
   - ✅ **الصواب**: يجب أن يستهدف الـ Scroll Engine (`ScrollToTop.jsx`) دائماً `document.getElementById('main-scroll-container')` ليقوم بتصفير التمرير وإرجاع المستخدم لقمة الصفحة بدقة.

2. **حصانة الجلسة (Offline-First Session Resilience)**:
   - عند تشغيل التطبيق والتحقق من حساب المستخدم في الخلفية:
   - ❌ **الخطأ**: تدمير الجلسة ومسح التوكن والبيانات المحلية ومطالبة المستخدم بتسجيل الدخول مجدداً لمجرد حدوث تأخير في الشبكة أو (Network Error) بسيط.
   - ✅ **الصواب**: لا تدمّر الجلسة إلا إذا أرجع الخادم صراحةً خطأ `401 Unauthorized` (نهاية الصلاحية). للشبكات الضعيفة، يتم فوراً الاعتماد على النسخة المخبأة `talabia-auth` لتمكين المستخدم من الدخول بلا انقطاع.

3. **التعبئة التلقائية المدمجة (Native Autofill Integration)**:
   - ❌ **الخطأ**: تصميم حقول نصية (Inputs) لتسجيل الدخول مبهمة وبدون وسوم تعريفية، مما يعطل مديري كلمات المرور (مثل Google Password Manager).
   - ✅ **الصواب**: يجب إلزامياً وضع الخصائص الثلاث (`id` و `name` و `autoComplete`) لجميع حقول المصادقة (على سبيل المثال: `autoComplete="email"` للبريد، و `autoComplete="current-password"` للسر).

## 🚫 8. تجنب هذه الأخطاء (Anti-Patterns)

- ❌ **لا تعدل مباشرة** على الملفات داخل `/var/www/`؛ التعديل يبدأ من المستودع دائماً.
- ❌ **لا تكرر الملفات**؛ استخدم المسارات النسبية الموحدة.
- ❌ **لا ترفع صوراً** ذات أحجام ضخمة دون ضغطها.
- ❌ **لا تترك ملفات `.bak` أو `.old`**؛ السيرفر يجب أن يظل "نظيفاً" تماماً.
